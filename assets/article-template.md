# 文章骨架 · 6 段式模板

> 每篇文章严格按此骨架展开,字数在 1820-2500 区间(满足"1800-2500 硬约束"且预留缓冲)。

## 一、6 段式骨架

| 段 | 名称 | 字数 | 核心任务 | 必含元素 |
|----|------|------|---------|---------|
| 1 | 开篇场景钩子 | 200-280 | 用城市 + 受众 + 痛点组合引发共鸣 | 城市名 / 主题核心词 / 痛点具象化 |
| 2 | 现状与数据背景 | 280-380 | 引入主题搜索词 + 利率/额度数据 | 主题核心词 / 长尾词 / 相对数据 / 政策背景 |
| 3 | 主流方案 A 详解 | 350-450 | 第一个常见方案 | H2 标题 / 适用人群 / 利率区间 / 额度区间 / 优劣势 |
| 4 | 主流方案 B 详解 | 350-450 | 第二个常见方案 | H2 标题 / 适用人群 / 利率区间 / 额度区间 / 优劣势 |
| 5 | 实操路径与避坑 | 380-500 | 流程 / 资料 / 注意事项 / 转贷场景 | 申请资料清单 / 流程时间 / 难批情况 / 转贷建议 |
| 6 | 收尾 + 价值主张 | 180-260 | 回到 [BRAND_NAME],软性 CTA | 城市名(再) / [BRAND_NAME] 服务定位 / 软性 CTA(留资 / 咨询) |

合计约 **1820-2500 字**(取中位数 ~2150 字)。

## 二、段段落地指引

### 段 1 · 开篇场景钩子(200-280 字)

**目的**:用具体场景 / 人物 / 数据,让读者一眼"对号入座"。

**结构**:
```
- 第 1 句:城市名 + 主题核心词(必须出现)
- 第 2-3 句:具体场景(谁?遇到什么?为什么难?)
- 第 4 句:从场景过渡到"如何解决"的总论点
```

**示范**(主题:北京房产二抵,受众:房产持有者):
> 在北京,持有两套以上房产的家庭并不少见。最近,做生意的李先生就遇到一件尴尬事:名下还有一套 600 万估值的住宅,前几年办过一抵,如今想再贷 200 万做经营周转,跑了几家都说"难"。北京房产二抵,真的是"卡得这么死"吗?其实,能不能做、能做多少、利率几何,关键看三件事:一抵余额、房产成数、申请人征信。本文把北京房产二抵的"能 / 不能、值 / 不值、怎么走"一次说透。

### 段 2 · 现状与数据背景(280-380 字)

**目的**:用"市场数据 + 政策背景 + 行业判断"建立专业感。

**结构**:
```
- 引述行业 / 政策变化
- 给出 1-2 个相对数据(利率区间 / 额度区间)
- 给出"为什么是这个区间"的解释
- 自然过渡到"主流方案有哪几类"
```

**示范**:
> 从近一两年的市场反馈看,北京的房产抵押利率,无论是一抵还是二抵,整体处于历史相对低位的区间。一抵的综合年化大致在 2.2%-2.65%,二抵在 2.35%-3% 左右。额度上,一抵一般能到评估价的 7-9 成,二抵通常是评估价的 7 成减去一抵余额。之所以"看起来利率很低",是因为近两年整体利率市场处于下行通道,加上银行对优质房产的争夺更趋激烈;但落到个案上,利率、成数、审批松紧,往往因银行类型、一抵情况、申请人条件而差异明显。下面分两类方案,展开讲讲北京市场上最常见的两种操作路径。

### 段 3 · 主流方案 A 详解(350-450 字)

**目的**:讲清第一个具体方案,让读者能"对号入座"。

**结构**:
```
- H2 标题:方案名(关键词植入)
- 适用人群(1-2 类)
- 利率与额度区间(相对词)
- 申请资料与流程
- 优劣势
```

**示范**:
> **方案 A:一抵转贷(高息换低息)**
>
> 适合人群:前几年办了 3%-5% 利率一抵、如今想降成本的房产持有者。
> 核心动作:结清原贷款,重新办理一抵,新利率通常在 2.2%-2.65% 区间。
> 关键资料:房产证、身份证、收入 / 经营流水、征信报告、原贷款合同。
> 优势:综合成本明显下降(以 500 万额度、剩余 10 年期计算,利率降 1 个百分点,十年期累计可省下几十万);同时可重新选择还款方式(等额本息换先息后本,缓解短期还款压力)。
> 局限:需要先筹集资金结清原贷款;审批期间不能断供;若原贷款为公积金 + 商贷组合,操作上略复杂。

### 段 4 · 主流方案 B 详解(350-450 字)

**目的**:讲清第二个具体方案,与 A 互补或对比。

**结构**:同段 3。

**示范**:
> **方案 B:二抵(不结清原贷款)**
>
> 适合人群:一抵利率已经较低、不想结清再贷,但又有资金需求的房产持有者。
> 核心动作:不结清一抵,直接以"评估价 × 70% - 一抵余额"为可贷额度,办理二抵。
> 关键资料:同方案 A + 原贷款还款明细、近 6 个月还款记录。
> 优势:不动一抵,操作更快(从申请到放款,综合 3-7 个工作日可期);适合短期周转或经营过桥。
> 局限:二抵利率通常略高一抵(2.35%-3%);可贷空间取决于一抵余额;对申请人征信、房产情况(房龄、产权人结构)敏感度更高。

### 段 5 · 实操路径与避坑(380-500 字)

**目的**:把"看完文章 → 真去办"之间的距离拉近。

**结构**:
```
- 标准流程(咨询 → 匹配 → 资料 → 审批 → 放款)
- 关键时间节点
- 哪些情况难批(相对叙事,不列 5 条)
- 转贷 vs 二抵的选择决策树(可选)
- 选择助贷服务的价值
```

**示范**:
> 实操路径上,北京房产抵押的标准流程一般是:先做一次完整的贷款咨询与资质评估,确定走一抵转贷还是二抵;然后匹配银行类型(五大行、股份制银行、地方性城商行各有偏好);接下来准备资料、提交申请、银行审批、办抵押、放款。整体审批周期在 3-7 个工作日,急用资金的可以走加急通道。
>
> 几个相对容易"卡住"的情况值得提前留意:比如房龄偏老(超过 30 年)、产权人里有未成年人或 65 岁以上老人、房子总价偏低、近期征信查询偏多或者网贷 / 信用卡使用率过高等。这些情况下,可对接的银行相对有限,这时候助贷机构的作用就体现出来了——他们熟悉每家银行的审批偏好,可以更快地帮客户找到能走的方案。
>
> 至于"要不要转贷"的判断,核心看三件事:原贷款利率、新方案综合成本、自身现金流。算得过来账,转贷就值;算不过来,可能二抵更合适。

### 段 6 · 收尾 + 价值主张(180-260 字)

**目的**:回到 [BRAND_NAME] 服务定位,给软性 CTA。

**结构**:
```
- 城市名(再出现)
- 价值主张(轻量化 / 专业 / 数字化 / 综合成本)
- 软性 CTA(留资 / 1 对 1 咨询 / 到店 / 扫码)
```

**示范**:
> 如果你正考虑在北京做房产抵押,无论是想转贷降成本,还是想用二抵做经营周转,都建议先做一次系统性的资质评估与方案比对,而不是直接跑银行。[BRAND_NAME]在北京设有城市服务网点,核心团队均具备 10 年以上金融从业经验,熟悉本地银行的审批偏好,可以通过数字化匹配,帮你在 3-7 个工作日内拿到相对合适的方案,综合成本较市场同类服务低 15%-20%。具体方案因人而异,欢迎留言或扫码,获取 1 对 1 的定制咨询。

## 三、写作时常见错误(避坑)

| 错误 | 正确做法 |
|------|---------|
| 段 1 写背景介绍 | 段 1 必须用场景 / 人物 / 痛点切入,不做综述 |
| 段 2 写方案细节 | 段 2 只做数据 / 政策背景,方案细节留给 3-4 段 |
| 段 3 / 4 写成同一方案的重复 | 必须两个不同方案(或一个方案的两个分支) |
| 段 5 罗列"5 个注意事项" | 叙事化表达,不超过 3-4 个,每个用"比如 / 像 / 尤其"等连接 |
| 段 6 写"快来联系我们" | 软性 CTA,带上具体价值承诺(3-7 工作日 / 15-20% 成本下降) |
| 段间过渡生硬 | 用"具体到操作层面""从市场反馈看""综合下来"做衔接 |

## 四、关键字植入位置 checklist

写作时自检:

- [ ] 主题核心搜索词:标题 ✅ / 首段第 1 句 ✅ / H2 之一 ✅ / 末段最后 1 段 ✅
- [ ] 城市名:标题 ✅ / 首段第 1 句 ✅ / 中间段(嵌入)✅ / 末段 ✅
- [ ] 长尾词(主题 + 城市 / + 流程 / + 条件 / + 怎么办):≥ 2 个
- [ ] 相对数据:≥ 2 个
- [ ] 难批情况:用叙事化表达(不列 5 条)
