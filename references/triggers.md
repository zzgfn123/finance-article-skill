# 触发词与意图识别

> 这一文件定义了 SKILL.md 第 0 步(解析输入 & 缺失追问)所需的所有判定规则。

## 一、激活触发词(命中任一即激活)

| 触发短语 | 触发动作 | 优先级 |
|---------|---------|--------|
| 「帮我写贷款文章」 | 启动完整流程,需询问主题/城市 | 高 |
| 「帮我写关于【城市】的文章」 | 启动流程,锁定城市,需询问主题 | 高 |
| 「帮我写发布文章」 | 启动流程,默认主题=企业融资,需询问城市 | 高 |
| 「帮我写 N 篇」 | 启动流程,篇数 = N(默认 5) | 高 |
| 「/loan-article-skill」前缀 | 强制激活 | 最高 |

## 二、参数提取(从用户输入解析)

### 主题(topic)
- 显式给出:`主题:XXX` / `关于XXX的文章` / `写XXX`
- 隐式给出:`房产抵押` / `企业经营贷` / `上班族信用贷` / `转贷` / `二抵` / `法拍房` / `商业房产抵押`
- 缺失:从 `assets/title-patterns.md` 给出 5 选 1 速选

### 城市(city)
- 显式给出:`城市:XXX` / `在XXX` / `XXX的` / 城市名直接出现
- 隐式给出:无,必须明确
- 缺失:从 `references/city-profiles.md` 给出 11 选 1 速选 + 自定义入口

### 篇数(count)
- 显式给出:`N篇` / `写N个` / `给我N个`
- 默认:**5**
- 范围:**1 - 10**(超过 10 建议分批,避免单次 token 压力过大)

### 标题(title)
- 显式给出:`标题:XXX` / `用这个标题:XXX` / 直接给标题
- 缺失:生成 5 个备选标题(参照 `assets/title-patterns.md`)

### 受众人群(audience)
- 显式给出:`针对XXX` / `面向XXX` / `受众是XXX`
- 隐式给出:从城市 + 主题自动推导(见 `references/city-profiles.md`)
- 缺失:不询问,直接由城市画像自动派生

## 三、缺失追问的标准话术

### 缺城市

```
请告诉我目标城市(可单选 / 多选):
  1) 北京  2) 上海  3) 深圳  4) 广州  5) 杭州
  6) 苏州  7) 南京  8) 武汉  9) 重庆  10) 成都
 11) 其他城市(直接输入,如:长沙 / 厦门 / 郑州)
```

### 缺主题

```
请告诉我文章主题(可单选 / 多选):
  1) 房产一次抵押(住宅)
  2) 房产二次抵押
  3) 企业经营贷 / 开票贷 / 纳税贷
  4) 上班族 / 个人信用贷
  5) 商业房产抵押(商铺 / 写字楼 / 厂房)
  6) 转贷(高息换低息 / 换还款方式)
```

### 缺标题(用户确认主题 + 城市后)

直接生成 5 个备选标题(见 `assets/title-patterns.md`),并标注角度类型。

### 缺篇数

不问,默认 5 篇。如果用户后补「再加 3 篇」,正常处理(但建议每 5 篇一个批次)。

## 四、歧义处理

### 「帮我写关于北京的文章」— 主题不明
- 行为:启动流程,先问主题
- 提示:「请问是写哪类主题?房产抵押 / 企业经营 / 个人信贷 / 转贷?」

### 「帮我写关于房产抵押的文章」— 城市不明
- 行为:启动流程,先问城市
- 提示:「请问目标城市?」+ 速选

### 「帮我写 N 篇」— 主题 + 城市都不明
- 行为:一次性问主题 + 城市(分两行)
- 提示:依次给两个速选菜单

### 「帮我写发布文章」— 全部不明
- 行为:默认主题 = 企业融资,只问城市
- 提示:「默认主题:企业经营贷。请问目标城市?」

## 五、负面触发(不激活的输入)

- 「贷款 / 借钱 / 怎么贷款」 — 单纯提问,不写文章 → 不激活
- 「帮我看一下这篇贷款文章」 — 已有文章,要求审稿 → 用 `simplify` / `code-review` 类技能
- 「写一篇非贷款的 SEO 文章」 — 主题不匹配 → 不激活
- 「写一篇关于银行的对比文章」 — 含具体银行名风险 → 提示换为类目表述,或拒绝并说明

## 六、并发与批量

- 用户给一次输入,但希望生成多组(不同城市或不同主题) → 支持,在一次调用里生成多 bundle
- 单次 token 预算紧张时,自动降级:5 篇 → 3 篇,先验证流程可跑通
